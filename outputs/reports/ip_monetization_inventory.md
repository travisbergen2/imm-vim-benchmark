# IP Monetization Inventory

This is a first-pass commercial triage of the uploaded IMM / RPCS-1 materials.

## High-Level Split

- `sell now`: productized offers with a clear buyer, a clear outcome, and a clear delivery path
- `lead magnet`: useful, credible, but better used to attract attention than to charge directly
- `authority content`: research and proof material that strengthens trust and can be sold later or packaged into premium releases
- `research-only`: material that is still too technical, too unsettled, or too theorem-heavy to sell directly

## What Can Be Sold Now

1. `RPCS-1 Agent Tuner`
   - Sell it as a productized service and SDK.
   - The current positioning is strong: it promises a diagnostic answer to a concrete agent problem.
   - The deck already contains a pricing ladder.

2. `rpcs1.dev` paid diagnostic
   - This is the cleanest immediate cash path.
   - The site already has pricing, a checkout handoff, and a brief intake.
   - Best fit: buyers who want a specific diagnosis before rollout.

3. `NL2Build Cloud`
   - This is the clearest software-product candidate in the new tranche.
   - It already has a product shape: natural language prompt in, Android app artifacts out.
   - Best fit: founders, solo builders, and small teams that want fast mobile MVP creation.

4. `IMM` bundle / textbook
   - Package `Volume I` plus the textbook assets as a premium digital bundle.
   - Use the volume series as upsells or chapter-separated editions.
   - Best fit: audience that wants a coherent framework instead of a single paper.

5. `Fractal Youniverse` / `Fractal Labs` pitch assets
   - Best treated as a high-authority offer stack and investor/partner narrative.
   - These should support the core products rather than replace them.

6. `RPCS1` technical bundle
   - Package the RPCS1 paper set as the technical justification layer behind the tuner.
   - Use it to sell credibility, not as a standalone mass-market item.

## What Should Not Be Sold Directly Yet

- RH theorem/proof manuscripts
- Yang-Mills deconstruction drafts
- raw formal proof strategy documents
- raw implementation internals like `nl2build_core.py`, `job_status_ui.py`, and utility scripts

These are better treated as:

- authority content
- lead-generation material
- future premium technical products

## Recommended Packaging Order

1. Sell `rpcs1.dev` as the immediate commercial offer.
2. Sell `RPCS-1 Agent Tuner` as the productized version of the same idea.
3. Bundle `IMM` volumes into a premium library or course.
4. Use RH/YM manuscripts to strengthen authority and attract a technical audience.
